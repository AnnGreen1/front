GCJ-02 坐标偏移

中国法律要求地图坐标经 GCJ-02 加密，而 Leaflet 用 WGS-84。直接用高德/腾讯瓦片会导致标记偏移 100~700m。

// 用开源库纠偏
import { gcj02ToWgs84 } from 'coordtransform'
const [lng, lat] = gcj02ToWgs84(gcjLng, gcjLat)
L.marker([lat, lng]).addTo(map)

或使用 leaflet-chinesetms 插件自动处理。

Canvas vs SVG 渲染器

默认 SVG 每个矢量 = 一个 DOM 节点，1000+ 标记严重卡顿。Canvas 渲染器适合大数据量：

// 全局 Canvas
const map = L.map('map', { preferCanvas: true })

// 或仅特定图层
L.circleMarker(latlng, { renderer: L.canvas() })

代价：失去 CSS 样式和 hover 事件，需用 pmousedown 等替代。

invalidateSize 陷阱

容器大小变化后地图不自动重算，出现灰色区域或瓦片错位。这是新手最常见的 bug：

// 侧栏收起/展开后
map.invalidateSize()

// 有 CSS 动画时
sidebar.addEventListener('transitionend', () => {
  map.invalidateSize({ animate: true })
})

// React 中
useEffect(() => {
  setTimeout(() => map.invalidateSize(), 100)
}, [sidebarOpen])
内存泄漏与正确销毁

SPA 切路由只移除 DOM 不销毁地图 → 事件监听器 + 定时器泄漏 → 内存持续增长。

// 正确顺序
map.eachLayer(l => map.removeLayer(l))
map.remove()

// React useEffect
useEffect(() => {
  const map = L.map(ref.current)
  return () => map.remove()  // 清理函数
}, [])
Pane 与图层遮挡

Leaflet 有 6 个内置 Pane，z-index 各不同。自定义 Pane 解决遮挡问题：

// 让 GeoJSON 在标记之上、弹窗之下
map.createPane('geoPane')
map.getPane('geoPane').style.zIndex = 450
// markerPane=600, popupPane=700

L.geoJSON(data, { pane: 'geoPane' })
Popup 异步内容

Popup 默认 300px 宽，异步内容需手动更新：

// 设置宽度
marker.bindPopup(html, { maxWidth: 400 })

// 点击后异步加载
marker.on('popupopen', async () => {
  const res = await fetch('/api/info')
  marker.getPopup().setContent(render(res))
})
