天地图服务接口

天地图提供丰富的在线 GIS 服务，全部免费（需申请 Key）

瓦片服务 URL 规则

天地图瓦片地址格式：

// 矢量底图
http://t{s}.tianditu.gov.cn/vec_w/wmts?tk=YOUR_KEY

// 矢量注记（标注层）
http://t{s}.tianditu.gov.cn/cva_w/wmts?tk=YOUR_KEY

// 卫星影像
http://t{s}.tianditu.gov.cn/img_w/wmts?tk=YOUR_KEY

// 卫星注记
http://t{s}.tianditu.gov.cn/cia_w/wmts?tk=YOUR_KEY

// 地形晕渲
http://t{s}.tianditu.gov.cn/ter_w/wmts?tk=YOUR_KEY

// 地形注记
http://t{s}.tianditu.gov.cn/cta_w/wmts?tk=YOUR_KEY

// {s} = 0~7  8 台服务器轮询

_w 后缀 = EPSG:3857（Web 墨卡托），_c = EPSG:4326（经纬度直投）。

地名搜索 API 
// GET 请求
https://api.tianditu.gov.cn/search?
  postStr={
    "keyWord":"故宫",
    "level":"12",
    "mapBound":"116,39,117,40",
    "queryType":"1",
    "start":"0",
    "count":"10"
  }
  &type=query
  &tk=YOUR_KEY

// 返回 JSON：名称、坐标、地址等
逆地理编码（坐标→地址） 
https://api.tianditu.gov.cn/geocoder?
  postStr={
    "lon":"116.3975",
    "lat":"39.9087",
    "ver":"1"
  }
  &type="geocode"
  &tk=YOUR_KEY

// 返回：formatted_address, addressComponent

  &type="drive"

// 返回：路线坐标点、距离、时间

与 Leaflet / OL 结合
Leaflet 接入天地图瓦片 
// 矢量底图 + 注记
L.tileLayer(
  'http://t{s}.tianditu.gov.cn/vec_w/wmts?tk=YOUR_KEY&SERVICE=WMTS&REQUEST=GetTile&VERSION=1.0.0&LAYER=vec&STYLE=default&TILEMATRIXSET=w&FORMAT=tiles&TILEMATRIX={z}&TILEROW={y}&TILECOL={x}',
  { subdomains:'01234567', maxZoom:18 }
).addTo(map)

// 注记叠加层
L.tileLayer(
  'http://t{s}.tianditu.gov.cn/cva_w/wmts?tk=YOUR_KEY&SERVICE=WMTS&REQUEST=GetTile&VERSION=1.0.0&LAYER=cva&STYLE=default&TILEMATRIXSET=w&FORMAT=tiles&TILEMATRIX={z}&TILEROW={y}&TILECOL={x}',
  { subdomains:'01234567', maxZoom:18 }
).addTo(map)

注意 URL 中的 {s} {z} {x} {y} 是 Leaflet 模板变量，需要手动拼完整 URL。

OpenLayers 接入天地图瓦片 
new ol.layer.Tile({
  source: new ol.source.WMTS({
    url: 'http://t0.tianditu.gov.cn/vec_w/wmts?tk=YOUR_KEY',
    layer: 'vec',
    style: 'default',
    matrixSet: 'w',
    format: 'tiles',
    tileGrid: new ol.tilegrid.WMTS({
      origin: ol.extent.getTopLeft(
        ol.proj.get('EPSG:3857').getExtent()
      ),
      resolutions: [...],
      matrixIds: [...]
    })
  })
})

OL 接入 WMTS 要手动配 tileGrid，比较繁琐。推荐用 ol-wmts-capabilities 插件自动解析。

必须申请 Key

天地图 API 必须在 控制台 注册并申请 Key。免费但有调用量限制：

  • 瓦片服务：无明确限制（合理使用）
  • 搜索/编码：每秒 30 次，每天 10000 次
  • 路径规划：每秒 30 次，每天 5000 次
坐标系：GCJ-02 偏移

天地图瓦片基于 EPSG:4326（WGS-84），但标注层和搜索结果是 GCJ-02 坐标。叠加使用时会有微小偏移。

天地图自己的 JS API 已内部处理，但你用 Leaflet / OL 接入时需要注意。

瓦片 URL 中的 _w 和 _c

vec_w = Web 墨卡托投影（EPSG:3857），适用于 Leaflet / OL / Mapbox

vec_c = 经纬度直投（EPSG:4326），适用于 OL 4326 模式

绝大多数场景用 _w。用错会导致瓦片完全加载不出来或偏到非洲。

WMTS 参数必须完整

天地图 WMTS 是严格的 RESTful 风格，参数一个都不能少：

http://t{s}.tianditu.gov.cn/vec_w/wmts?
SERVICE=WMTS
&REQUEST=GetTile
&VERSION=1.0.0
&LAYER=vec
&STYLE=default
&TILEMATRIXSET=w
&FORMAT=tiles
&TILEMATRIX={z}
&TILEROW={y}
&TILECOL={x}
&tk=YOUR_KEY

少了任何一个参数都会返回错误或空白瓦片。

卫星图和标注是两个图层

天地图的卫星影像是裸图（没有路名、地名），标注在另一个图层（cia）。必须手动叠加：

// 影像底图
const imgLayer = ...    // img_w
// 注记叠加
const labelLayer = ...  // cia_w
// 两个都加到地图上

这和高德/百度不同——它们的卫星图自带标注。

HTTPS 和 HTTP 混用问题

天地图同时支持 HTTP 和 HTTPS，但部分子域名 HTTPS 证书有问题。

建议统一用 HTTPS：

// 推荐
https://t{s}.tianditu.gov.cn/...

// 不推荐（混合内容警告）
http://t{s}.tianditu.gov.cn/...
