坐标系统：EPSG:3857 vs 4326

OpenLayers 内部用 EPSG:3857（Web 墨卡托），所有坐标都要转换：

// 经纬度 → OL 坐标（必须转！）
import { fromLonLat } from 'ol/proj'
const pos = fromLonLat([116.40, 39.90])
// [12953935.0, 4852023.0] — 不是经纬度了！
// OL 坐标 → 经纬度
import { toLonLat } from 'ol/proj'
const [lng, lat] = toLonLat(olCoord)

忘记 fromLonLat 是新手第一大坑，标记会跑到非洲去。

View 独立于 Map

Leaflet 的视图控制在 map 上，OL 要通过独立的 View 对象：

// 创建 View
const view = new ol.View({
  center: ol.proj.fromLonLat([104, 35]),
  zoom: 5
})

// View 的方法
view.animate({ zoom: 12, duration: 1000 })   // 动画
view.setCenter(ol.proj.fromLonLat([121, 31])) // 跳转
view.setRotation(Math.PI / 4)                 // 旋转
Feature 必须被 Source 包裹

OL 是四层结构，比 Leaflet 多两层：

// OL 四层：Map → Layer → Source → Feature
new ol.Map({
  layers: [
    new ol.layer.Vector({              // Layer
      source: new ol.source.Vector({   // Source
        features: [
          new ol.Feature({             // Feature
            geometry: new ol.geom.Point(ol.proj.fromLonLat([116, 39]))
          })
        ]
      })
    })
  ]
})

// Leaflet 两层：Map → Marker
L.marker([39, 116]).addTo(map)

OL 更啰嗦但更灵活——一个 Source 可以被多个 Layer 引用。

弹窗用 Overlay，不是 Popup

OL 没有内置 Popup 类，用 ol.Overlay 挂载 DOM 元素：

const popupEl = document.getElementById('popup')
const overlay = new ol.Overlay({
  element: popupEl,
  positioning: 'bottom-center',
  offset: [0, -10]
})
map.addOverlay(overlay)

// 点击时设置位置
map.on('click', e => {
  overlay.setPosition(e.coordinate)
  popupEl.querySelector('.desc').textContent = e.coordinate
})
样式函数 vs 静态样式

静态样式所有 feature 一个样，样式函数可以按属性不同显示不同：

// 样式函数：根据属性动态返回样式
vectorLayer.setStyle((feature) => {
  const value = feature.get('value')
  return new ol.style.Style({
    fill: new ol.style.Fill({
      color: value > 80 ? 'rgba(34,211,153,0.3)' : 'rgba(249,115,22,0.3)'
    }),
    stroke: new ol.style.Stroke({ color: '#22d3ee', width: 1.5 }),
    text: new ol.style.Text({
      text: feature.get('name'),
      fill: new ol.style.Fill({ color: '#e5e7eb' }),
      font: '13px Noto Sans SC'
    })
  })
})
